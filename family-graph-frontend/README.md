# Архив родства — Vue-фронтенд на v-network-graph

Визуализация генеалогического дерева из вашего GraphQL API (`schema.graphql` /
`resolvers.ts`): персонажи — узлы графа (или строки списка), связи
father/mother/spouse — рёбра. Клик по узлу/строке открывает карточку
человека. Есть формы создания и редактирования персонажа (`createPerson` /
`updatePerson`).

## Запуск

```bash
npm install
npm run dev
```

Откройте адрес, который выведет Vite (обычно `http://localhost:5173`).

При первом запуске в верхней панели «API» введите URL вашего GraphQL endpoint
(например `http://localhost:4000/graphql`) и нажмите «Подключиться» — адрес
сохранится в `localStorage` браузера. Альтернативно можно скопировать
`.env.example` в `.env` и задать `VITE_GRAPHQL_URL` — он будет использован как
значение по умолчанию.

## Откуда берутся данные карточки персонажа

Карточка человека **не использует `personWithRelations`**. Вместо этого:

1. По клику на узел/строку вызывается `person(id: ID!)` — он уже отдаёт
   `mother_obj` / `father_obj` / `spouse_obj` (вложенные `Person`),
   `surname_obj`, `nationality_obj`, `birth_place_obj`, `death_place_obj` и
   т.д.
2. Детей сервер не возвращает (в `Person` такого поля нет). Поэтому
   composable (`useGenealogy.ts → computeChildren()`) сам ищет их среди уже
   загруженного списком `persons` — фильтром по полям `mother`/`father`,
   указывающим на текущий `id`. Эти ID есть в каждом `Person`, поэтому
   дополнительных запросов к серверу не требуется.

Это и есть смысл фразы «искать в `Person(id)` и по найденным id находить
нужную информацию»: сервер отдаёт только прямые связи (ID и один уровень
вложенных объектов), а обратные связи (кто чьи дети) выводятся на клиенте по
уже загрученному списку персон.

## Два представления дерева

Переключатель «Граф / Список» в шапке:

- **Граф** (`v-network-graph`) — узлы цветом по полу, пунктир у умерших,
  раскладка по поколениям, рёбра father/mother + золотая пунктирная линия
  супругов.
- **Список** — то же дерево в виде вложенного списка: каждый персонаж — это
  строка, рядом (через «⚭») показан супруг, дети — с отступом ниже.
  Строится той же логикой, что и граф (`useGenealogy.ts → buildFamilyTree()`),
  но из соображений читаемости списка ребёнок привязывается только к одному
  родителю (приоритет — отцу, если он есть в дереве, иначе матери), чтобы не
  показывать его дважды.

Оба представления используют общую панель карточки персонажа справа и общий
`selectPerson()`.

## Создание и редактирование

- «+ Добавить человека» в сайдбаре — создаёт персонажа в выбранном дереве
  (`createPerson`, требует `genealogy_tree_id`).
- «Редактировать» в карточке — открывает форму с предзаполненными данными
  (`updatePerson`). Поле `id` человека не меняется; в `input` передаются
  `mother_id` / `father_id` / `spouse_id` (как и ожидает резолвер).
- «Удалить» — `deletePerson` с подтверждением.
- Выбор отца/матери/супруга — выпадающий список из персонажей текущего
  дерева (исключая самого редактируемого человека).
- Место рождения/смерти упрощено до даты (+ "приблизительно") и
  страна/город — без улицы/дома/квартиры. При необходимости легко расширить
  `PersonForm.vue` + `api/types.ts` дополнительными полями `street/house/
  apartment`, аналогично `BirthPlaceInput` в `schema.graphql`.
- Поле «человек умер(ла)» управляет наличием `death_place` в запросе — сам
  флаг `isalive` пересчитывает резолвер на backend.

## Структура проекта

```
src/
  api/
    types.ts      — TS-типы, отражающие schema.graphql
    client.ts      — fetch-обёртка + хранение GraphQL endpoint
    queries.ts     — все GraphQL-запросы/мутации (включая person(id))
  composables/
    useGenealogy.ts — загрузка данных, построение nodes/edges для графа,
                       buildFamilyTree() для списка, computeChildren() для
                       карточки персонажа, CRUD-действия
  components/
    EndpointBar.vue       — настройка адреса API
    TreeSidebar.vue       — список деревьев, создание дерева, легенда графа
    FamilyGraphCanvas.vue — обёртка над <v-network-graph>
    TreeListView.vue       — корень списочного представления
    TreeListNode.vue       — рекурсивная строка списка (персона + супруг + дети)
    PersonPanel.vue        — карточка персонажа
    PersonForm.vue         — форма создания/редактирования
  App.vue
```

## Известные ограничения / куда расширять

- Запрос `persons` в схеме не принимает фильтр по дереву — фильтрация по
  `genealogy_tree_id` происходит на фронтенде (`useGenealogy.filteredPersons`).
  Для больших деревьев имеет смысл добавить на backend
  `persons(genealogy_tree_id: ID)`.
- Раскладка графа по поколениям и список дерева — простые эвристики (без
  идеального учёта пересечений рёбер/повторных браков при больших деревьях).
- Адреса (улица/дом/квартира) в форме не редактируются — см. раздел выше.
